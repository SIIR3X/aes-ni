Encrypts or decrypts data using AES in OFB mode.

OFB mode uses AES encryption to generate a keystream, which is XORed with the input. This function works for both encryption and decryption since the process is symmetric.

Padding is not required. The IV must be 16 bytes and should be unique for each message.

Parameters
ctxPointer to a valid AES context (initialized with aes_context_init).
iv16-byte initialization vector (IV). Must not be NULL.
inputPointer to the input data (plaintext or ciphertext).
input_lenLength of the input data in bytes.
outputPointer to the output buffer (ciphertext or plaintext). Must be at least input_len bytes.
